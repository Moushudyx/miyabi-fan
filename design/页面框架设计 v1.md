# 页面框架设计(2026-02-20 第一版)

这个网站是一个二次元角色(星见雅, 来自游戏《绝区零》)的爱好者网站

响应式设计, 主要分为页顶(页眉)与主体两个部分

1. 整体需要兼容 PC 端与移动端(响应式设计)
2. 页顶部分包含 Logo 区域与导航栏区域, 固定在顶部
3. 主体部分仅包含内容展示区, 页面跳转动作由浮动按钮触发

添加路由功能, 整体设计为 SPA(单页应用), 包含首页、关于页等多个页面

- 目前想先部署在 GitHub Pages 上, 需要使用 Hash 模式路由, 之后如果部署在支持 History 模式的服务器上再切换

页面上存在多处需要获取数据的地方(展示优秀同人作品等), 需要隔离代码与数据

## 首页设计

整体采用滚动吸附设计, 每个部分占据一个视窗高度, 通过滚动或点击浮动按钮进行页面切换(需要适配移动端手势滑动、浏览器端滚动条)

整体布局抽象为一个 PageSlider 组件, 内部包含多个 PageSection 组件, 每个 PageSection 组件占据一个视窗高度

- 考虑使用 scroll-snap 来实现滚动吸附效果, 需要兼容移动端
- 考虑到完全使用 CSS 制作响应式会导致代码难以理解, 可以在 PageSection 上使用插槽来实现不同页面的内容, 但是需要注意插槽内容的样式需要在组件外部定义, 以保持组件的通用性

PageSection 第一次进入可视区域时, 子元素触发动画效果, 之后不再触发

- 考虑使用 Intersection Observer 来检测, 需要兼容移动端
- 动画效果希望是陆续滑动淡入(不是同时出现), 考虑使用 CSS 或者 Anime.js 实现

### 第一屏

第一屏想放角色立绘, 最好是放上动态立绘

- 考虑到性能等问题, 先不考虑使用 WebGL/Lottie 动画(没有相关素材, 当然,,, 我也不会), 先考虑动态图片或者(循环播放的)视频

### 第二屏

第二屏放上同人作品, 设计是横向的层叠图片的模式, 鼠标划到哪一张, 哪张就展开, 点击跳转对应作品页面

特殊动效: 收起时展示一张图片, 展开时展示动图/截短的视频

示例: 默认展开第一张, 页面大致样式是

```
| 图片1(展开, 占据很大的宽度) | 图片2(收起) | 图片3(收起) |
```

此时鼠标划到图片2上, 图片2展开, 图片1收起, 图片3保持收起

```
| 图片1(收起) | 图片2(展开, 占据很大的宽度) | 图片3(收起) |
```

抽象为一个 FanWorksGallery 组件, 内部包含多个 FanWorkItem 组件, 每个 FanWorkItem 组件接受一个缩略图和一个插槽, 鼠标划过时展示插槽内容, 鼠标离开时展示缩略图

- 展开后在左下角展示一些作品信息(标题、作者等), 可以再添加一个信息层插槽, 只在展开时显示
- 需要区分移动端的划过与点击事件, 移动端划过时不展开, 点击时展开
- 考虑到完全使用 CSS 制作响应式会导致代码难以理解, 可以在 FanWorkItem 上使用插槽来实现不同作品的内容, 但是需要注意插槽内容的样式需要在组件外部定义, 以保持组件的通用性
- 屏幕很窄时(比如移动端), 改为纵向的; 信息层改到右上角

### 第三屏

一些官方消息, 比如游戏官方商店、官方周边等, 设计为双层录像带架子

上层放置官方消息, 下层放置相关链接(商店、周边等)

特殊动效: 需要实现点击从架子上抽出一盒录像带的效果(架子上展示侧面, 抽出后一个 3D 动画转动录像带到正面, 同时全屏背景虚化, 点击虚化的背景可以放回录像带), 抽出后点击跳转对应页面

特殊动效: 需要实现反光效果, 即录像带转动时不同的面会有不同的反光, 可以使用 CSS 的动画或者使用 Anime.js 实现

抽象为一个 VideoTapeRack 组件, 内部包含多个 VideoTapeItem 组件, 每个 VideoTapeItem 组件接受一个侧面图和一个插槽, 点击时正面展示插槽内容, 再次点击时收起

- 页面分两层所以是两个 VideoTapeRack 组件
- VideoTapeRack 组件可以横向滚动, 以适配移动端
- VideoTapeItem 居中, 理论上不用做移动端适配, 只要限制宽高不在小屏幕上太大/在大屏幕上太小
- 插槽的部分可以是图片或者动图、视频等, 收起动画结束后不渲染正面内容以节省性能(仅在用户能看到的时候渲染)

## 技术

框架: Vue 3 + TypeScript + Vite

样式: SCSS

动画函数库: Anime.js

工具函数库: foreslash

### 技术细节

样式处理: 使用 SCSS 预处理器, 但是不使用 Vue 的 scoped 样式

代码文件:
- 组件文件: src/components/ 目录下, 每个组件一个文件夹, 包含 .vue 文件, 样式写在组件文件中
  - 示例: src/components/PageSlider.vue, src/components/FanWorksGallery.vue
- 页面文件: src/pages/ 目录下, 每个页面一个 .vue 文件, 样式写在组件文件中
  - 示例: src/pages/Index.vue, src/pages/About.vue
  - 页面文件子组件: src/pages/<页面名称>/ 目录下, 每个子组件一个 .vue 文件, 样式写在组件文件中
    - 示例: src/pages/Index/HeroSection.vue, src/pages/Index/FanWorksSection.vue
- 路由文件: src/router/index.ts, 定义路由配置
- 数据文件: src/data/ 目录下, 每个数据类型一个文件, 导出对应的数据对象
- 工具函数文件: src/utils/ 目录下, 每个工具函数一个文件, 注意不要和工具函数库撞车
  - 比如 shuffle 直接从 foreslash 导入, 不要自己写一个 shuffle 函数放在 utils 里
